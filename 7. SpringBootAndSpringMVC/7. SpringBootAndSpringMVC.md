# Understanding Spring Boot and Spring MVC

## What is a web app?

**An app access by the web browser is a web app**

One web app is composed of two parts:
* **The client side** is responsible for the interaction between the user and the web app. One web browser is responsible for *receiving and sending responses to the web server* as well as *provide a way for the user to interract with the app*. This side can be also be referd as the **frontend**
* **The server side** is responsible for receiving request from the client and send responses. Other responsiblities the server has is to *implement logic that process and stores the client's request*. This side can also be refered as the **backend**

There are two approaches when creating web apps:
* Apps where **the backend provides the fully prepared view in response to a client's request**. *The browser directly interprets the data received from the backend and displays this information to the user*
* Apps where **frontend-backend seperation is used**. The backend only sends **raw data to the front end** and the **browser runs a separate frontend app that process the backend responses**

### Using a servlet container in web app development

The web browser communicates with the server side using HTTP, **this protocol describes how the client and server exchange data over the network**

**Servlet container** is a Jva object that translate HTTP messages, using them *you dont have to take care for the implementation of the communication layer*

## The Magic of Spring Boot

*To create a Spring web app, you need to create a servlet container and create a servlet instance and configurate it inside the servlet container, this caused a lot of headaches*

**Spring Boot is a tool for implementing modern Spring apps**, it helps with **creating Spring apps more efficiently and focus on the business code by eliminating a huge part of configurating the code**

Spring Boot features:
* Simplifiend project creation
* Depedency starters, *Spring Boot groups certain dependencies used for a specific purpose with dependency starters*
* Autoconfiguration based on dependencies

### Using dependency starters to simplify the dependency management

Dependency starters **save you plenty of time**, it is basically a **group of dependencies added to configurated a project to a specific purpose**

*In the past you had to add all the dependencies for web appliction with Spring and you had to check are their versions compatible with each other which took time*
*But with spring-boot-starter-web everything is stored together with compatable versions and it eases the configuration process*

## Implementing a web app with Spring MVC

To add a web page to your app, you have to follow those steps:
* Write a HTML document with the content you want to be display by the browser
* Write a controller with an action for the web page 

*In order to start this example, generate from [spring.io](https://start.spring.io/index.html) project with Spring Web dependency, after its creation run the main method to validate is it working*

All of the static HTML pages that are going to be display by the browsers are found in **resources/static** folder, since Spring Boot expects there to find the pages to render
```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Home Page</title>
</head>
<body>
    <h1>Welcome!</h1>      
</body>
</html>
```

Now, we need to create a **controller**, a **controller is a component of the web pp that contains methods executed for specific HTTP request**
*In this current example, the controller will return a reference of the html page we just created*

In order **to create a controller**, we will use the ```@Controller``` annotation, *remember Spring will also add a bean of this class to its context*
**If we want the browser to display the html page when a specific path is accessed**, we have to use ```@RequestMapping```
```java
@Controller
public class MainController {

    // when the client access the /home path the html page is going to be returned
    @RequestMapping("/home")
    public String home() {
        return "home.html";
    }
}
```

When starting the app, **by default TomCat starts the server on localhost:8080** so in order to access the html file, go to **localhost:8080/home**

![](https://image2url.com/r2/default/images/1772700605999-46e9e384-16ea-4c8e-b2be-bfeb3e531e3e.png)

Spring has a set of components that interacts with each other and there is a background process that follows these steps:
* The client makes an HTTP request
* Tomcat gets the client's HTTP requests and sends the requests to the dispatcher servlet
* Since the dispatcher servelt is the entry point of the Spring web app, its **responsibility is to manage the request further** and **it has to find what controller action to call for the request and what to send back as a response**. *This dispatcher servlet is also known as front controller*
* The front controller has to find which controller action (methods defined in ```@Controller``` class) to call, for this task it uses **handler mapping** component to find controller action associated with the request with ```@RequestMapping``` annotation
* After finding which controller action to call, the method is called and returns the needed page name, this page is reffered as **the view**, if no actions were found for the request **HTTP 404 NOT FOUND** status is returned
* After receiving the page name the dispatcher servlet finds the actual page file and sends it as a response, **in order to get the view content from the pagethe front controller uses component named View Resolver**
* The dispatcher servlet returns the rendered view in the HTTP response

As you can see Spring **considerably simplifies the development of a web app by arranging the setup**. The only thing we need to do **is to write controller actions and map them to requests using annotations**
*A large part of the logic is hidden in the framework, this helps you to write apps faster and cleaner*
