# Using the Spring web scopes

As you can remember, you can choose to declare a bean s one of the following:
* **Singleton**, here **Spring uniquely identifies each instance with a name in the context**
* **Prototype**, here **Spring only manages the type and creates a new instance of that class every time someone requests it**

In web application, there are **different kind of web scopes**:
* **Request scope**, here **Spring creates an instance of the bean class for every HTTP request** and *this instance exists only for that specific HTTP request*
* **Session scope** here **Spring creates an instance and keeps the instance in the server's memory for the full HTTP session** and *Spring links the instnace in the context with the client's session*
* **Appliction scope**, here  **the whole instance is unique in the app's context**

To practice and see how those different scope works we will create a multi-step exercise

## Using the request scope in a Spring web app

Remember that **Spring creates a new instance for every HTTP request, any new HTTP requests creates and uses different instance of the same class**
Spring **manages the bean type and uses it to get instances for each new request**

In order to start with the exercise we will create a new project and add the dependencies ```spring-boot-starter-thymeleaf``` and ```spring-boot-web-starter```

*Basically the app will consist of asking the user for credentials and depending are they right or wrong specific message will be outputted*
*In order to show the user what happened, we need to implement a page and  controller class*

*For your curiosity the HTML page that the client will see looks like this, and it is stored in ```resources/templates```*
```html
<!DOCTYPE html>
<html lang="en" xmlns:th="http://www.thymeleaf.org">            
<head>
    <meta charset="UTF-8">
    <title>Login</title>
</head>
<body>
    <form action="/" method="post">                               
        Username: <input type="text" name="username" /><br />
        Password: <input type="password" name="password" /><br /> 
        <button type="submit">Log in</button>                      
    </form>
    <p th:text="${message}"></p>                                  
</body>
</html>
```
*To display the message with the logic's result, a parameter is needed to be send from the controller to the view*
*The HTML is created in such a way to send a POST request to the server the moment a client has entered its logic credentials*
![image](https://image2url.com/r2/default/images/1773072460100-925f7699-bc88-46dd-8f11-02dd613bfb39.png)

In order to implement a login logic we have to create an object
```java
@Component
@RequestScope
public class LoginProcessor {

    private String username;
    private String password;

    public boolean login() {
        String username = this.getUsername();
        String password = this.getPassword();

        if("natalie".equals(username) && "password".equals(password)) {
            return true;
        } else {
            return false;
        }
    }

    // getters and setters
}
```
We use ```@RequestScope``` to make this bean's scope as request only, since **the information the client uses is valuable we do not want to store it internally**

The controller will look like this:
```java
@Controller
public class LoginController {

    @GetMapping("/")
    public String loginGet() {
        return "login.html";
    }

    @PostMapping("/")
    // This method is called from the HTML definition when the user enters some credentials and takes the request parameters
    public String loginPost(@RequestParameter String username, @RequestParameter String password, LoginProcessor loginProcessor ,Model model) {

        // Spring will automatically create an instance of the type LoginProcessor from the context and inject it into the method
        loginProcessor.setUsername(username);
        loginProcessor.setPassword(password);

        boolean isLoggedIn = loginProcessor.login();

        if(isLoggedIn) {
            model.addAttribute("message", "You are logged in");
        } else {
             model.addAttribute("message", "Login failed");
        }
    }
}
```

## Using the session scope in a Spring web app 

*The example in the book's chapter were unnecessary complex, so they are not covered much*

**A session-scoped bean is an object managed by Spring and creates an instance for HTTP session**. **Once the client sends a request, the server reserves a place in the memory for this request, for the whole duration of the session.**
*The instance can be reused for the same client as long as the session is active, the data stored in the session-scoped bean can be accessed by the client's request*

**Basically a session-scoped bean allows to store data shared by multiple requests of the same client**

**In order to create session-scoped bean you can use ```@ScopedBean```**

## Using the application scope in a Spring web app

*The author doesn't recommend to use application scope*

**The instance of an application-scoped bean is shared by all the HTTP requests from all clients**. *This is very similar on how singleton works*
