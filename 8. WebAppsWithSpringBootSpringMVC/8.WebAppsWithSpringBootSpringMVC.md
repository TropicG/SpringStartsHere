# Implementing web apps with Spring Boot and Spring MVC

**A templete engine is a dependency that allows to get and display easily data the controller sends**

## Implementing web apps with a dynamic view

Before starting the implementation, lets remember what was the MVC flow when receiving and sending HTTP requests:
* The client sends HTTP request to the server
* The dispatcher servlet uses the handler mapping to find out what controller action *(remember this is a method annotation with ```@RequestMapping```)* to call
* The dispatcher servlet calls the controller's actions
* After executing the action, the controller returns the view name the dispatcher servlet needs to render into the HTTP response
* The response is sent back to the client

![image](https://image2url.com/r2/default/images/1772783009706-017b925c-acdb-4601-93b1-e7e2d5c2ae52.png)

The whole idea right now is to **make the controller send the view name to the view as well**. **This view will incorporate this data to define the HTTP response**

*This view resolver component is responsible to visualize the raw data send from the controller*

*In this exercise we will add Thymeleaf dependency, it is a template engine that allows to easily send data from the controller to the view and display data*

Creating the controller, we map one method to */home*
```java
@Controller
public class MainController {

    @RequestMapping("/home")
    // The parameter stores the data that will be send to the view resolver
    public String home(Model page) {
        // for simplicity, accept that here we add the values with unique names that will be displayed by the view
        page.addAttribute("username", "Katy");  
        page.addAttribute("color", "red");      
        return "home.html";   
    }
}
```

*The view returned from the method needs to be located in resources/templates since this is not a static view, but a dynamic view*  

The HTML file would look like this
```html
<!DOCTYPE html>
"here is just like adding a dependency to maven"
<html lang="en" xmlns:th="http://www.thymeleaf.org">  
    <head>
      <meta charset="UTF-8">
      <title>Home Page</title>
    </head>

    <body>
      <h1>Welcome
      "Here we refer to the data sent by the controller"
      <span th:style="'color:' + ${color}"
            th:text="${username}"></span>!</h1>               
    </body>
</html>
```

So, when the client goes to */home* he would see this on his browser

#### Getting data on the HTTP request

There are multiple ways on how the client **can send data to the server with HTTP requests**:
* **HTTP request parameter** represents a way to send values from the client in a key-value pair format. To send HTTP request with parameters **they have to be appended to the URI in a request**. *They are also called query parameters and are used for sending small quantity of data*
* **HTTP request header** is similar to the HTTP request parameter, **the parameters doesn't appear in the URI, but in the Headers of the HTTP request**. *They are still used for small quantity of data*
* **Path variable sneds data through the request path itself, it is the same as request parameter approach**. *It still needs to be used for small quantity of data*
* **HTTP request body**, it is used **to send large quantity of data**

#### Using request parameters to send data from client to server

Request parameters shall be used in the following scenarios:
* **The quantity of data is not large**
* **Optional data needs to be send**. *A request parameter is a clean way to deal with a value the client might not have sended*

This is example on how you could use request parameters
![image](https://image2url.com/r2/default/images/1772787034096-a02da444-83e9-4c9c-9b94-3ca0693d287f.png)

Here is an example on how to get the request parameter from the request in the controller
```java
@Controller
public class MainController {
    @RequestMapping("/home")
    public String home(@RequestParam String color, Model page) {
        page.addAttribute("username", "Katy");
        page.addAttribute("color", color);         
        return "home.html";
    }
}
```
In order to get the request parameter, you need **to add ```@RequestParam``` to the parameter that will hold the value**
*```@RequestParam``` tells Spring it needs to get the value from the HTTP request parameter with the same name as the method's parameter name*

To send the dat with the HTTP requests, you can do it like that
```http://localhost:8080/home?color=blue```

When using HTTP request parameter, **you need to extend the path with ? followed by pairs of key=value parameters seperated by &**

For example if you need to send multiple parameters:
```http://localhost:8080/home?color=blue&name=Jane```
*Keep in mind, you need to add a second parameter to hold the value for name*
![image](https://image2url.com/r2/default/images/1772787725167-9d6ec1e5-f9cf-451a-8e34-8c54f2f5351f.png)

### Using path variables to send data from client to server

Instead of using HTTP request parameter, you **directly set variable values in the path**

Here is the comparison: ```http://localhost:8080/home?color=blue``` / ```http://localhost:8080/home/blue```, *value is not identified with a key*
*The server side extract the value from the path's specific position*. *More than one parameters could be used, but avoid using more than a couple*

To refer a path variable in a method, you **give it a name and add it to the path between curly braces**. After that you have to use ```@PathVariable``` to get the path variable
```java
@Controller
public class MainController {
    // {color} path variable represents the value provided in the path
    @RequestMapping("/home/{color}")
    // the name of the path variable annotated with @PathVariable, gets the value from the path
    public String home(@PathVariable String color, Model page) {
        page.addAttribute("username", "Katy");
        page.addAttribute("color", color);
        return "home.html";
    }
}
```

For example, if you want to display the text with red color, you can use ```http://localhost:8080/home/green``` or ```http://localhost:8080/homo/blue```

## Using the GET and POST HTTP methods 

**HTTP GET** method only **retrieves data by obtaining it from the server and doesn't change data in any way**

There are multiple HTTP methods:
* **GET**, *the client request only receives data*
* **POST**, *the client request sends new data to be added by the server*
* **PUT**, *the client's request changes a data record on the server side*
* **PATCH**, *the client's request partially changes a data record on the server side*
* **DELETE**, *the client's request deletes data on the server side*

*In the following example we are going to create a list of products, in which the user can add or remove products*
*Dont forget to add ```spring-boot-starter-thymeleaf``` and ```spring-boot-starter-web``` depedencies to your project*

```java
// This class will describe the products in our exercise
package model;

public class Product {
    private String name;
    private double price;
    // getters and setter
}
```

The list of products have to be stored somewhere and the business logic must be implemented so we are creating a **ProductService class**
```java
package service;

@Service
public class ProductService {

    private List<Product> products = new ArrayList<>();

    public void addProduct(Product p) {
        products.add(p);
    }

    public List<Product> findAll() {
        return products;
    }
}
```

Let's create a controller that accept the client's requests and **fulfill the use cases to retrieve and add products**, this shall be done by **calling the Service layer**
Typically we use ```@RequestMapping``` to execute client's request, but we are going to use ```@GetMapping``` and ```@PostMapping``` for different approach
*Typically when using ```@RequestMapping``` by default it is going to catch only HTTP GET request if it is not configurated* 
```java
package controller;

@Controller
public class ProductsController {
    
    @Autowired
    private final ProductService productService;

    // This method is going to be executed when the client's request is HTTP GET
    @GetMapping("/products")
    // We use a Model parameter to send the data from the controller to the view
    public String viewProducts(Model model) {
        var products = productService.findAll();
        model.addAttribute("products", products);

        return "products.html";
    }

    // This method is going to be executed when the client's request is POST
    @PostMapping("/products")
    // We use a Model parameter to send the data from the controller to the view
    public String addProduct(@RequestParam String name, @RequestParam double price, Model model) {
        Product p = new Product();
        p.setName(name);
        p.setPrice(price);
        productService.addProduct(p);

        var products = productService.findAll();
        model.addAttributes("products", products);

        return "products.html";
    }
}
```
Analogical there are ```@PutMapping```, ```@DeleteMapping```, ```@PatchMapping``` for every HTTP method 

*If you are curious how does the HTML looks like you could observe it, it must be places in ```resources/templates``` directory*

*Since Spring is famous for omitting code, you could pass the ```@RequestParameter``` for the ```@PostMapping```, take a look at this example*
```java
@PostMapping("/products")
// You could directly skip @RequestPrameter and use Product object as a parameter if the request parameter's names are the same as the Product class attribute's names
// Spring will know how to match them and automatically creates the object 
public String addProduct(Product p, Model model) {
    productService.addProduct(p);

    var products = productService.findAll();
    model.addAttribute("products", products);

    return "products.html";
}
```
